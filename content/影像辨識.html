影像辨識

簡報

周昀融 設計的 影像辨識期末告

影片

1.

動機：虎尾鎮曾經因為多次垃圾分類沒有分類好，導致垃圾被停運好幾次，其實有時候我自己也很懶得分類，我希望可以製作一個可以自動分類垃圾的垃圾桶。

預期目標：這次我會先以生活中常見的垃圾進行辨識，我希望他可以分辨常見的寶特瓶、鐵鋁罐、紙類，在和實體機構進行配合。

環境保護趨勢： 全球資源回收需求日益增加，但人工分類成本高且容易出錯。

技術結合： 結合深度學習（YOLO）與電腦視覺（OpenCV），實現自動化資源回收系統。

應用場景： 智慧垃圾桶原型。透過鏡頭自動辨識垃圾種類，導引至正確的回收桶。

2.

模型架構

  • 使用的模型： YOLOv8 Nano (YOLOv8n)。

  • 選擇原因： 模型輕量化、推論速度快（約 1.8ms），非常適合部署在資源有限的硬體設備（如智慧垃圾桶的嵌入式系統）。

1. 骨幹網路 (Backbone) —— 負責「看特徵」

這是模型的第一部分，就像人的眼睛和視神經。

功能： 它的工作是從原本的圖片中提取特徵。它會把圖片不斷縮小，提取出線條、形狀、紋理，直到變成高維度的特徵圖。

YOLOv8 的技術： 使用了 CSPDarknet 結構。

白話文： 它負責看出一張圖裡哪裡是圓的、哪裡是紅色的、哪裡有邊緣。

2. 頸部 (Neck) —— 負責「整合資訊」

這是模型的中間部分，就像神經傳導系統。

功能： 它負責把 Backbone 提取到的「淺層特徵（線條）」和「深層特徵（物體輪廓）」融合在一起。因為小物件需要看細節，大物件需要看輪廓，Neck 把這些資訊混合，讓模型能同時看清楚大大小小的東西。

YOLOv8 的技術： 使用了 PAN-FPN (Path Aggregation Network)。

白話文： 它確保模型不會因為看大東西就忽略小東西，把不同尺寸的特徵圖疊加起來。

3. 頭部 (Head) —— 負責「做決定」

這是模型的最後部分，就像大腦的決策區。

功能： 根據 Neck 給的資訊，最終畫出框框 (Bounding Box) 並判斷它是什麼類別 (Class)。

YOLOv8 的技術： 使用了 Decoupled Head (解耦頭)。意思是它把「畫框框」和「分類別」這兩件事分開來做，比較不會互相干擾。

白話文： 它最後輸出三樣東西：

位置 (xywh)：框框在哪裡？

類別 (Class)：是 Paper 還是 Can？

信心度 (Confidence)：我有幾成把握？

3.使用工具及技術

第一階段

  • 訓練環境： Google Colab (使用 T4 GPU 加速)。

  • 訓練參數： Epochs=50, Imgsz=512。

  • 推論環境： 本地 Windows 電腦、Miniconda 虛擬環境、OpenCV 影像串接。

第二階段

  • 訓練環境： PVE虛擬機 (使用 GTX1070 GPU )、python 1.31.1、cuda124。

  • 訓練參數： Epochs=50, Imgsz=640。

  • 推論環境： 本地 Windows 電腦、Miniconda 虛擬環境、OpenCV 影像串接。

4.資料集準備

第一階段

  • 資料來源： TACO Dataset（開源垃圾資料集）。

  • 資料標註與合併： 使用 Roboflow 平台將 60 種細項分類合併為三大類：Plastic_Bottle (寶特瓶)、Can (鐵鋁罐)、Paper (紙類)。

  • 資料增強 (Augmentation)： 加入旋轉 (Rotation)、模糊 (Blur) 等技術，模擬垃圾桶內複雜的光線與拍攝角度。

第二階段

  • 資料來源： roboflow Universe中搜尋到的 Plastic Bottle 2.0 Computer Vision Model

  • 資料標註與合併： 這次我決定先訓練寶特瓶的資料集

  • 資料增強 (Augmentation)： 加入旋轉 (Rotation)、模糊 (Blur)

第二階段：重新來過

1.數據收集、數據處理

我在查看robo flow的過程中，我發現taco的數據集大部分是壓扁，形狀、外觀都沒有那麼原始的垃圾，所以第二階段我決定先專注訓練寶特瓶的模型。

首先我在Roboflow Universe 上面尋找到了一個寶特瓶的訓練集，總共3700多張照片，我把它fork回來我的workspace。

fork回來後我調整了一下預處理 (Preprocessing)，我設定了以下兩項

  • Auto-Orient: 開啟（確保手機拍照的角度不會讓模型困惑）。

  • Resize: 設定為 640x640 (Stretch)。這是 YOLO 系列最標準的輸入大小。

接下來我對照片進行數據增強 (Augmentation)，以下是我的設定

  • Horizontal Flip: 開啟。

  • Rotation: 設在 -15° 到 +15° 之間（模擬手持傾斜）。

  • Brightness: 設在 -15% 到 +15%（模擬不同天氣/室內光線）。

  • Blur: 少量（模擬相機晃動或對焦不準）。

3.接下來進入訓練環節

from ultralytics import YOLO


# 確保這行路徑是完全正確的
final_yaml = "F:/code/bottle_dataset/data.yaml"

# 再次確認檔案真的在那裡
import os
if os.path.exists(final_yaml):
    print("✅ 檔案確認存在")

# 建議先用 cpu 測試一下路徑是否通暢，沒問題再換 device=0 (GPU)
model = YOLO('yolov8n.pt')

results = model.train(
    data=final_yaml,  # 確保這是一個字串，例如 "F:/code/bottle_dataset/data.yaml"
    epochs=50,
    imgsz=640,
    batch=24,
    device=0,        # 如果報 CUDA 錯誤，請先改為 'cpu'
    name='bottle_exp_1'
)
