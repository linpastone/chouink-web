周昀融 設計的 資料庫管理系統

簡報PDF

講稿PDF

🌟 絕對重點：交易管理 (Transactions) 與 ACID 屬性

在交易處理中，必須確保以下 ACID 屬性，這部分是考試的關鍵：

  • 原子性 (Atomicity)：不可分割，交易中的操作必須「一起成功或一起失敗」。
    •
  • 一致性 (Consistency)：確保每次交易後的資料一致性，這是交易設計器 (Transaction designer) 的責任，設計者不需擔心併發和系統故障的問題。
    •
  • 隔離性 (Isolation)：可以變形（先後順序改變）但不能同時執行。這意味著執行過程不一定是嚴格串行的，而是可序列化的 (Serializable)。
    •
  • 持久性 (Durability)：確保交易完成後資料永久保存。
    •
  • 備註：TP monitor 主要負責確保原子性、持久性與特定級別的隔離性，提供無故障且非併發的抽象環境。

🌟 絕對重點：外來鍵與完整性限制 (Integrity Constraints)

  • 處理外鍵違規 (Handling Foreign Key Violations)：您的筆記明確指出這是「考試必考 1-50」。
    •
  • 外來鍵 (Foreign Key) 定義：若關聯 R1 中的屬性 A1 參照 R2 中的屬性 A2，則只要 A1 有值，R2 中就必須有對應的元組其 A2 值相同，且 A2 必須是 R2 的候選鍵 (Candidate key)。
    •
  • 參照完整性 (Referential integrity)：上述 A2 必須是候選鍵的規定即為參照完整性；若 R2 中不存在對應的列，即屬違反參照完整性。
    •
  • 完整性限制 (Integrity constraints)：您的筆記特別標註「簡報要有」。

💡 重點一：實體關聯圖 (ER Diagram) 與資料屬性

  • 實體 (Entity)：指我們希望儲存資訊的人、地點、事件或事物。
    •
  • 關聯 (Relationship)：實體之間的連結，關係本身也可以被視為一種特殊的實體。
    •
  • 屬性 (Properties)：實體的特徵，可以是簡單或複雜的。
    •
  • 計算屬性設計：有些資料（例如 GPA）不需要實際儲存，而是在需要時計算出來即可。
    •
  • ER圖拆分設計：例如「訂單」，在設計上可以被拆成「訂單主檔案」以及「訂單明細」

💡重點二：關聯式資料模型 (Relational Data Model) 基礎

  • 發展背景：由 Codd 在 1970 年代提出，是資料庫歷史上最重要的創新，基於邏輯與數學。
    •
  • 結構對應：數學上的「關係 (Relation)」對應到資料庫的資料表 (Table)；「元組 (Tuple)」對應到列 (Row)；「屬性 (Attribute)」對應到欄 (Column)
    •
  • Tuple 特性：元組的排序是無關緊要的 (Tuple ordering immaterial)，順序改變不影響資料表意義。
    •
  • 操作 (Operations)：運算子可以從現有的資料表衍生出新的資料表，分為一元（如刪除特定列）與二元（如聯集、笛卡兒積）運算，這些數學特性能用來判斷等價性，對查詢最佳化 (Query Optimization) 非常重要。

💡 重點三：資料庫系統架構 (三層抽象結構)

資料實際是以位元(bits)儲存，但為了方便使用，資料庫將其分為三個抽象層級，各自擁有獨立的 Schema（結構描述）：

  1. 外部層 (External Level)：支援單一或多群使用者，不同使用者需要不同的視觀表 (Views)，通常透過 SQL Views 實作。
    2.
  2. 概念層 (Conceptual Level)：全域的邏輯表示法，由所有使用者共用，代表資料的底層意義，是資料庫設計的基礎，透過概念 DDL (Conceptual DDL) 實作。
    3.
  3. 內部層 (Internal Level)：實體層 (Physical layer)，包含區塊 (Blocks)、分頁 (pages)、I/O，與硬體相關，包含 Hash、Heap、B-tree 等資料結構及指標 (pointers)。
    4.
  4. 資料獨立性 (Data Independence)：資料庫分離了邏輯與實體資料表示法，讓應用程式可以修改，而不需要改變底層資料結構。

💡 重點四：資料庫應用與系統組成

  • 系統組成：資料庫系統由 Data、Hardware、Software、Users 組成。
    •
  • 資料庫使用情境：
    • OLTP (線上交易處理)：例如紀錄日常行為，「買了3罐湯和1盒餅乾，更新資料庫以反映此事件」。
      •
    • OLAP (線上分析處理)：大範圍查詢，例如「去年冬天在東北地區所有門市，有多少顧客同時買了湯和餅乾？」。
      •
    • Data Mining (資料探勘)：尋找潛在模式，例如「顧客常一起購買的食物有沒有什麼有趣的組合？」。

💡 重點五：SQL 標準與權限控制

  • 標準演進：SQL-92 是現代 SQL 的基礎，隨後的 SQL:1999 增加了遞迴查詢、物件導向等特性，SQL:2003 增加了 XML 與 Window functions 相關特性。
    •
  • 權限控制 (GRANT)：可授權的操作包含 SELECT, INSERT, DELETE, UPDATE, REFERENCES。例如可以特別指定僅對特定欄位（如 Grade）授予 UPDATE 權限，但在標準 SQL 中不能對單一欄位限制 SELECT，不過可以透過其他方式模擬。
